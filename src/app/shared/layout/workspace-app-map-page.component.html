Application map

How the Nexir Core workspace is structured: routes, layers, and where responsibilities live.

arrow_back Back to our view
  • / — Landing; redirects to the app when you are already signed in.
  • /workspace-map — This orientation page (available before and after sign-in).
  • /login, /register — Guest-only authentication flows.
  • /app/* — Authenticated product shell: sidebar, toolbar, and feature outlets.
  • /app/dashboard — Session summary, quick actions, recent entities.
  • /app/entities — Searchable catalog; star rows for Favorites; row opens the editor. Create and edit use mock persistence (localStorage).
  • /app/favorites — Table of starred entities (same star toggle; row opens edit).
  • /app/account — Change password (mock user registry in this browser).
  • /app/settings — Appearance: theme (system / light / dark) and UI density.
core
Session, storage keys, guards, theme, validators, notifications, favorite ids (local mock state).
shared
Layout shell, landing, application map, reusable UI blocks (cards, badges, empty states).
features
Vertical slices: auth, dashboard, entities, favorites, settings, account — routes, pages, stores/APIs per feature.

Users and passwords (mock), session, preferences, entity rows, recent opens, and favorite ids are stored in localStorage for this browser profile only. There is no backend in this workspace.
